## Overview

QzPlatform is an online assessment system designed to manage computerized tests, examinations, and result processing for multiple user roles including administrators, instructors, and students.

I worked on the backend development of the platform, focusing on API design, authentication, and database structure, while also assisting in frontend integration.

## Key Features

- **Multi-Role System Architecture**: Designed backend logic to support Admin, Test Creator, and Test Taker roles, each with distinct permissions and workflows.
- **RESTful API Development**: Built structured API endpoints for authentication, user management, test creation, question handling, and result processing.
- **Authentication & Authorization**: Implemented secure login systems using JWT, including role-based access control for protected routes.
- **Test & Question Management**: Developed backend logic for creating tests, managing question banks, applying test settings (time limits, attempts), and handling submissions.
- **Automated Result Processing**: Structured systems for evaluating test submissions and returning scores and detailed results.
- **Database Design & Modeling**: Contributed to schema design using MongoDB, modeling entities such as users, tests, questions, attempts, and results.
- **Frontend Integration Support**: Assisted in connecting frontend dashboards with backend APIs, ensuring consistent data flow and usability.

## Technologies Used

- **Node.js / Express**: For building backend APIs and handling server-side logic.
- **MongoDB**: For database design and schema modeling.
- **JWT (JSON Web Tokens)**: For authentication and session management.
- **REST APIs**: For structured communication between frontend and backend systems.

## Challenges and Learnings

One of the main challenges was designing a backend structure that could handle multiple user roles with different permissions while keeping the system scalable and maintainable.

Structuring APIs in a way that cleanly separates concerns between authentication, test management, and user workflows required careful planning. Additionally, designing a flexible database schema for tests, questions, and attempts helped improve how data flows through the system.

This project strengthened my understanding of backend architecture, API design patterns, and real-world system organization.

## Outcome

The backend system provides a structured foundation for an online assessment platform, supporting core features such as test creation, user management, and result processing.

It demonstrates practical experience in building multi-role systems, designing APIs, and structuring backend applications for real-world use cases.
